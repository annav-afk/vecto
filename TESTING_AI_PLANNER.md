# Тестирование AI-планировщика

## Быстрый тест

### 1. Перейдите на страницу создания плана

Откройте приложение и нажмите "Создать план" или перейдите на `/goal-input`.

### 2. Заполните форму

**Пример 1: Быстрый проект**
- **Цель:** Создать лендинг для SaaS-продукта
- **Дедлайн:** Через 30 дней (1 месяц)
- **Часов/нед.:** 15ч

**Ожидаемый результат:**
- AI рассчитает: 30 дней × (15ч/7) = ~64 часа доступно
- Показывается сообщение: "AI рассчитает план на **30 дней** с доступными **64 часами**"

**Пример 2: Долгосрочный проект**
- **Цель:** Выучить Python с нуля до Junior-разработчика
- **Дедлайн:** Через 180 дней (6 месяцев)
- **Часов/нед.:** 20ч

**Ожидаемый результат:**
- AI рассчитает: 180 дней × (20ч/7) = ~514 часов доступно
- План будет более детальным и распределённым

### 3. Нажмите "Создать план"

AI начнёт генерацию. Вы увидите:
- Анимацию с Томи
- Шаги генерации (6 шагов)
- Автоматический переход на страницу плана

### 4. Проверьте результат

На странице плана:

#### ✅ Проверьте блок "План сгенерирован AI"
Должен отображаться над прогресс-баром с информацией:
- Часов в неделю
- Количество дней
- Доступное время

#### ✅ Проверьте задачи
Задачи должны быть:
- **Конкретными**: не "Работать над дизайном", а "Создать мокапы главной страницы и 3 экранов"
- **С временем**: каждая задача должна иметь оценку часов
- **Выполнимыми**: задачи разбиты на маленькие шаги

#### ✅ Проверьте подзадачи
Каждая задача должна содержать 2-4 подзадачи с конкретными действиями.

### 5. Проверьте консоль браузера

Откройте DevTools (F12) → Console.

Должны быть логи:
```
[Vecto AI] Plan via edge function — 4 phases
Plan generated for goal: "..." — 4 phases, 64h available
```

## Тестовые сценарии

### Сценарий 1: Малое время
- Цель: Написать статью
- Дедлайн: 7 дней
- Часов/нед.: 5ч
- **Результат:** План должен быть простым, 2-3 фазы, минимум задач

### Сценарий 2: Среднее время
- Цель: Запустить интернет-магазин
- Дедлайн: 60 дней
- Часов/нед.: 20ч
- **Результат:** План из 4-5 фаз, детальные задачи

### Сценарий 3: Большое время
- Цель: Создать мобильное приложение с нуля
- Дедлайн: 180 дней
- Часов/нед.: 30ч
- **Результат:** Подробный план, много задач, реалистичное распределение

### Сценарий 4: Очень малое время (стресс-тест)
- Цель: Запустить стартап
- Дедлайн: 14 дней
- Часов/нед.: 40ч
- **Результат:** AI должен упростить план или предупредить

## Что проверять

### ✅ Конкретность задач
- [ ] Задачи содержат конкретные действия
- [ ] Нет абстрактных формулировок
- [ ] Каждая задача понятна без контекста

### ✅ Временные оценки
- [ ] У каждой задачи есть оценка часов
- [ ] Сумма времени всех задач близка к доступному времени
- [ ] Задачи распределены равномерно по фазам

### ✅ Структура
- [ ] 4-5 фаз (этапов)
- [ ] 3-5 задач в каждой фазе
- [ ] 2-4 подзадачи в каждой задаче

### ✅ Реалистичность
- [ ] План выполним за указанное время
- [ ] Задачи логически упорядочены
- [ ] Фазы идут в правильной последовательности

## Ошибки и фолбеки

### Если AI недоступен
Система автоматически переключится на mock-генератор.

**Как проверить:**
1. Отключите интернет
2. Попробуйте создать план
3. План всё равно должен быть создан (из шаблонов)

### Если AI вернул ошибку
В консоли будет:
```
[Vecto AI] Edge function failed, using local mock generator
```

## Логи для анализа

### Успешная генерация
```
[Vecto AI] Plan via edge function — 4 phases
Plan generated for goal: "Создать лендинг" — 4 phases, 64h available
```

### Фолбек на mock
```
[Vecto AI] Edge function timed out after 25s
[Vecto AI] Edge function failed, using local mock generator
[MockAI] Generated plan for goal "..." — type: product, 6 phases
```

## Дополнительные проверки

### 1. Демо-режим
Протестируйте в демо-режиме (без регистрации):
- Перейдите на вкладку "Демо" в AuthModal
- Введите имя
- Создайте план
- Всё должно работать так же

### 2. Разные типы целей
Проверьте генерацию для разных категорий:
- Продукт (запуск бизнеса)
- Обучение (курсы, навыки)
- Фитнес (похудение, марафон)
- Контент (блог, YouTube)
- Карьера (поиск работы)

### 3. Облачная синхронизация
Если у пользователя есть токен:
- План должен сохраниться в облако
- Лимит использования должен увеличиться

## Известные ограничения

1. **Timeout 25 секунд**
   - Если AI не ответит за 25 секунд, используется mock-генератор

2. **Max tokens 6000**
   - Очень сложные планы могут быть обрезаны
   - AI предупредит об этом

3. **Fallback на mock**
   - При любой ошибке AI система использует локальные шаблоны
   - Качество может быть ниже, но план будет

## Обратная связь

После тестирования проверьте:
- [ ] Все планы генерируются
- [ ] Задачи конкретные и выполнимые
- [ ] Время распределено реалистично
- [ ] Интерфейс показывает информацию об AI
- [ ] Фолбеки работают
